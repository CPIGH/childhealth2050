# content/  ·  the website's editable text

Human-written content lives here so section owners can edit it without touching code. Full guide:
[docs/CONTENT-EDITING.md](../docs/CONTENT-EDITING.md).

## Layout

```
content/
├── site/         landing-page copy, About, shared blurbs   (Markdown .md)
├── profiles/     standardized/  → per-country context (.md, AI-generated first, then authored)
│                 deepdives/     → optional richer country deep dives (.docx)
│                 rubrics.md, _TEMPLATE.docx, deepdives.json (generated)
└── indicators/   description & methodology note per indicator (Markdown .md)
```

The Country Profiles list is generated by the publish workflow. Authors can add or update profile
content without editing an index by hand.

Two formats, on purpose:

- **Country profiles and deep dives** should stay structured and easy to review. The main profile page
  can begin with standardized, AI-generated draft text and later be revised by authors.
- **Site copy and indicator notes** remain Markdown (.md) for lighter text and explanatory notes.

## Rules of thumb

- Keep the main profile text short, structured, and clearly labeled when it is an AI draft.
- Keep deep dives as separate, richer authoring pieces when they exist.
- Keep the folder each file belongs in — pages look for content by path.
- The file in content/ is the **single source of truth** — don't keep a second copy elsewhere.
- Images referenced from text go in [assets/img/](../assets/img/), not here.
