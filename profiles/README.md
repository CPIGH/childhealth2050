# Country Profiles  ·  `/profiles`

**Tool 1 of 4.** A per-country scorecard — the single-country "home base" a user lands on to
understand one country at a glance.

## What it shows

- A **scorecard** across the four lenses from the concept note:
  - **Child Health Indicator(s)** — mortality, nutrition, coverage
  - **Equity** — gaps by wealth, sex, urban/rural
  - **Financing**
  - **Future Outlook**
- A short **"What makes this country unique?"** narrative (editable text, not a chart).
- Entry points into the other tools scoped to this country (compare it, explore its trends,
  see its 2050 trajectory).

## Content & data

- Narrative text: [`content/profiles/`](../content/profiles/) — one **Word document** per country
  (e.g. `nigeria.docx`), rendered in the browser via mammoth.js. See
  [content editing](../docs/CONTENT-EDITING.md).
- Numbers: fetched on demand from [`assets/data/`](../assets/data/) (small, rounded, split files).

## Link sharing

Selection lives in the URL query string, e.g. `/profiles/?country=nigeria`, so a copied link
reopens the same country. Each country view sets its own Open Graph tags for nice social previews.
See [docs/ARCHITECTURE.md](../docs/ARCHITECTURE.md).

## How the country list is built

A static site can't list a folder at runtime, so the page reads
[`content/profiles/profiles.json`](../content/profiles/) — a manifest regenerated by
[`tools/build-profiles-index.py`](../tools/build-profiles-index.py) at publish time. It scans the
folder for `*.docx`, skips `_`-prefixed files, and derives each link from the file name (label =
file name, `slug` = url-safe id). Authors just drop a `.docx`; the link appears on republish.
